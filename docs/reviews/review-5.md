## Game representation and network architecture

### Entity graph + explicit adjacency + D6 symmetry

You already encode Catan as an entity graph of tiles, nodes, edges, players, and context tokens, but your transformer uses dense attention without graph bias, and actions never directly attend to the board entities they affect. Catan RL work and general graph‑based RL show that **explicitly encoding adjacency and spatial relations improves learning efficiency**. - **Adjacency‑masked attention**: for most layers, restrict attention so each token primarily attends to neighbors and a global context token, plus a small set of “long‑range” slots, similar to graph transformers. ### Action/board cross‑attention

Currently, actions are scored by cosine similarity between an action embedding and the CLS token, meaning “build settlement at node 23” never directly looks at node‑23’s features. Catan‑specific networks and MCTS w/ heuristics often **bind actions to the local board region**. [arxiv](https://arxiv.org/abs/2008.07079)

I’d:

- Represent each action as a small set of **references to board tokens** (e.g., target node, adjacent edges, relevant hexes).
- Let the action head **cross‑attend to those specific tokens**, then combine that local information with the global CLS, so scoring “settlement at node 23” explicitly uses node‑23’s token, neighboring hexes, and player state. [arxiv](https://arxiv.org/abs/2008.07079)

This should reduce value noise and make placement decisions easier.

### Scaling the net

### Opponent pool integration

As above, integrate the opponent pool, borrowing from PFSP and league training:

This league+window+PFSP combination links your system to the broader self‑play literature’s recommendations for robustness. [arxiv](https://arxiv.org/pdf/2305.11476.pdf)

Once Catan‑Zero 2.0 is stable and leading in two‑player no‑trade, I’d extend:

- **Multi‑player, trading Catan**: here ReBeL and Student‑of‑Games become more relevant, since multi‑party negotiation and imperfect information make belief modeling and continual re‑solving more valuable.
