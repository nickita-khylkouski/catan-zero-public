# A1 training-signal root audit — 2026-07-16

## Executive conclusion

The historical result commonly described as “training on n128/n256 did not
work” was not a clean test of coherent n128 distillation.

The failed learner combined several different learning operators:

- fresh and historical PIMC-era policy targets;
- a legacy objective that mixed the sampled trajectory action into the MCTS
  target;
- terminal outcomes generated by different continuation-policy mixtures;
- independently normalized base and policy-AUX objectives whose optimizer dose
  was accounted as though they were one objective;
- value evidence reported primarily as state rows even though one terminal
  outcome was repeated across all rows of a game.

The evidence therefore does **not** establish that coherent n128 data lacks an
improvement signal. It establishes that the previous learner objective was not
one coherent policy-improvement operator.

## Root failures fixed on `main`

### 1. Sampled-action contamination can no longer reactivate silently

`tools/a1_one_dose_train.py` previously defaulted a missing
`policy_target_blend_semantics` field to `legacy_interpolate_v1`. A malformed or
historical recipe could therefore optimize:

```text
0.9 * CE(MCTS policy) + 0.1 * CE(sampled trajectory action)
```

without the launch making that semantic change explicit.

The final command boundary now requires a supported, explicit semantic value.
Current coherent training binds `policy_target_fallback_v2`, which uses the
complete search distribution when present.

Commit: `fbd63b0`

### 2. Value evidence now reports games, not only repeated rows

Terminal outcome supervision has one independent outcome per completed game,
not one independent outcome per decision row. The learner report now includes:

- clean terminal value rows;
- independent terminal games;
- row labels per independent outcome;
- repeated-label fraction;
- per-game value-weight effective sample size;
- rows and value mass per game;
- contradictory terminal outcomes within a game.

This prevents a million-row value corpus from being described as a million
independent value observations.

Commit: `c0b994d`

### 3. Whole-game validation can no longer be row-capped

Explicit validation seed ranges selected complete games and then applied
`validation_max_samples` by randomly sampling individual rows. The generic
random game split had the same post-split row-cap path.

That recreated the exact within-game validation pathology the game-level split
was intended to prevent. Explicit ranges now require an uncapped exact holdout.
Random game validation applies the cap while choosing complete games and may
slightly exceed the requested row count rather than split a game.

Commit: `52d6c06`

### 4. Policy dose now accounts for the real base + AUX objective

The actual loss is:

```text
policy_loss_weight * (
    independently_normalized_base_policy_mean
    + policy_aux_loss_weight * independently_normalized_AUX_policy_mean
)
```

The prior LR-area ledger collapsed base and AUX presence to one boolean. A
base-only, AUX-only, and base-plus-AUX update could spend the same recorded
dose even though their objective coefficients differed.

Dose metering now:

- tracks DDP-global base and AUX presence separately;
- includes `policy_aux_loss_weight`;
- scales the final boundary update by the complete configured objective;
- coefficient-weights realized effective policy mass.

Row exposure remains a diversity/reuse measure and is no longer confused with
optimizer objective strength.

Commit: `e4691ee`

### 5. The obsolete autonomous loop cannot execute

`tools/selfplay_loop.py` still encoded:

- n64 PIMC/determinization generation;
- the provenance-poor generic merge;
- mixed continuation-policy value labels;
- gateless promotion.

It remains importable for forensic command reconstruction, but direct execution
now fails immediately. Production iteration must use the current sealed A1
generation, authenticated composite, scratch learner, and promotion
transaction.

Commit: `acb81fa`

### 6. Scratch policy training is pinned to one exact teacher operator

The post-wave builder already attaches one versioned policy-target identity to
every fresh policy component. The scratch launcher did not explicitly pass the
accepted digest to `train_bc`, leaving a generic fail-open path for uniformly
unbound legacy corpora.

Scratch verification now resolves the exact identity over the declared policy
distillation component scope, requires all active components to carry it, and
passes it explicitly to the learner.

Commit: `37091e8`

## What the historical failed learner actually trained

The July-14 composite admitted four policy/value lineages:

- current producer;
- recent history;
- hard negative;
- historical replay.

Historical replay contributed about one fifth of the policy-active draws and
used a separately sharpened stored policy. The old learner also used
`soft_target_weight=0.9`, which injected sampled-action CE. Terminal outcomes
from all four continuation-policy regimes were admitted to the scalar value
head.

That learner was therefore not estimating one value function and not
distilling one search operator. Its internal improvement cannot be interpreted
as a clean coherent-n128 result.

The current post-wave composite has the right structural reset:

- policy: fresh same-operator components only;
- value: current-producer self-play outcomes only;
- historical replay: state/reanalysis evidence, not direct policy or value
  supervision.

## Remaining high-impact research blockers

### A. The value query distribution is not yet validated

Coherent search stops when control leaves the root actor. Long-horizon strength
therefore depends heavily on the value network at actor handoff, roll, and turn
boundary states.

The strongest existing calibration artifact evaluates active unforced policy
rows, not the states where search most often calls the value function. A
turn-boundary/evaluator-query holdout must be materialized from complete games
and reported separately from policy-decision rows.

Required metrics:

- bias and RMSE;
- Pearson/Spearman ranking correlation;
- calibration/ECE;
- per-phase and per-legal-width results;
- game-bootstrap confidence intervals;
- search uplift using the same fixed root set.

### B. Opening value remains weak

Existing evidence shows only roughly 0.15–0.19 correlation and about 0.99 RMSE
at opening/wide-action states. Increasing n does not solve a leaf evaluator
that carries little long-horizon information.

Do not blindly blend the stored root value: current opening root values are
also weak. The correct target experiment is targeted multi-seed,
multi-belief Monte Carlo or coherent reanalysis at opening and turn boundaries.

### C. The shared-representation trust region is incomplete

`trunk_lr_mult` and policy/value gradient-cosine telemetry cover the state
trunk. The current value path also consumes action representations created by
the action encoder, target gather, static residual, and action cross modules.

Those modules are policy/value-shared but are not part of the current “trunk”
trust-region surface. A future control must explicitly cover the full
config-dependent shared representation, and telemetry must distinguish:

- state trunk;
- action representation;
- private policy tower/readout;
- private value tower/readout.

### D. Global clipping couples new heads to the mature policy

All objectives are summed before one global gradient-norm clip. A large fresh
auxiliary/categorical/belief gradient can shrink every policy and trunk update
and alter their Adam moments.

New objectives must be commissioned by measured shared-gradient norm, with
clip scale and pre/post-clip contribution reported. Head-only warmup or scoped
clipping should be tested before enabling a large new objective jointly.

### E. Scalar tanh-MSE has a bad confident-error gradient

The deployed scalar value is `tanh(raw)`. Training MSE on that squashed output
multiplies the correction by the tanh derivative, so confidently wrong
predictions receive a vanishing gradient.

A matched causal arm should train binary win BCE on logits `2 * raw`, since:

```text
(tanh(raw) + 1) / 2 == sigmoid(2 * raw)
```

Search can retain the same tanh readout while the learner receives a
non-vanishing correction on confidently wrong examples. This is an experiment
to commission, not yet a production default.

### F. The scratch optimizer schedule remains intentionally unauthorized

The current scratch contract uses warmup 250. At checkpoint step 128 this is
only about 33 full-LR-equivalent updates, versus about 78.5 for the historical
warmup-100 trajectory. Early scratch checkpoints must not be compared as
matched historical doses.

The contract correctly remains `go_authorized=false` until LR, warmup, value
routing, and objective dose are deliberately commissioned.

## Next implementation order

1. Build evaluator-query/turn-boundary value calibration from whole games.
2. Add bias/correlation/ECE and game-bootstrap validation metrics.
3. Add a binary-win-BCE scalar value objective as an explicit experimental
   semantic.
4. Add a full shared-representation LR/gradient telemetry surface.
5. Add scoped/head commissioning for new objectives before joint training.
6. Commission LR/warmup/value routing using identical scratch initializers and
   the same coherent corpus.
7. Only then authorize the first large scratch training run.
