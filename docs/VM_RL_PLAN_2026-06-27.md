## Long-Term Algorithm Ladder

1. Gate the current value-rollout repair wave before launching more PPO. If
   `s9865`-`s9873` clear `value_rollout` but regress against `jsettlers_lite`,
   use `rollout_guard_score_repair`; if they still smear rollout pressure
   across tactical action classes, use `tactical_rollout_guard_repair`; if
   they clear both, escalate the best checkpoint to g12/g24 before making it a
   parent.
2. Launch `rollout_guard_score_repair` next, one VM first. It blends
   heuristic/JSettlers resource-plan score targets with value-rollout search
   score targets. The success criterion is not aggregate win rate; it must
   avoid heuristic, `jsettlers_lite`, `catanatron_value`, and `value_rollout`
   regression.
3. If rollout-guard still fails tactical legs, use explicit tactical
   action-prior gating before PPO. The first implementation is
   `tactical_rollout_guard_repair`, which chooses the action class from
   baseline/JSettlers scores and applies rollout pressure mainly within that
   class instead of smearing score pressure over every legal action.
4. Use VRPO-style Expected-SARSA/Q targets as critic variance reduction only
   after a branch passes strict tactical gates. Previous Q-policy mixing caused
   regressions; do not let critic improvements steer the actor without a hard
   non-regression gate.
5. Add DAGS-style intermediate starts before another broad PPO fanout. Use
   saved midgame states from strong/keeper games and rejected near-misses,
   start short self-play segments from those states on VMs, and train only
   under the same strict non-regression gates. This targets Catan's sparse
   terminal signal without changing the public promotion rule.
6. Move to ExIt/Student-of-Games style reanalysis once we have a stable
   non-regressing parent: generate value-rollout/search targets from saved
   intermediate states, distill them into policy/value heads, then continue
   PPO from the distilled checkpoint.
7. Build a population scheduler only after there are multiple non-regressing
   policies. Use payoff logs to select parents/opponents in a PSRO/PFSP style;
   do not maintain a population of policies that already fail
   `jsettlers_lite` or `value_rollout`.
8. Representation work is a later lane: graph/history/cross-dimensional
   Catan encoders are plausible, but they should be compared through the same
   strict gates and should not replace tactical repair work until the current
   candidate architecture stops failing rollout tactics.

## Algorithm Critique

Current broad PPO fanout is not the best use of compute. Most early `s97xx`
branches tie the champion exactly on common seeds, so they are not learning a
robust new behavior. The high-risk pieces are:

- Q advantage is untrusted until Q is calibrated. Chosen-action Q can be
  trained while legal-action Q for unchosen actions remains unconstrained. The
  `s9750`/`s9751` VRPO-style lanes support this: they improved some surface
  legs but failed `value_rollout`.
- Teacher anchors should not overwrite the critic with teacher-policy returns.
  Use `--anchor-value-coef 0.0` unless there is direct evidence otherwise.
- Pure self/value/heuristic opponent branches are too easy to overfit and
  should be recycled after stale checkpoints are pulled.
- Promotion gates must include tactical pressure. `jsettlers_lite` and
  `value_rollout` are the current cheap substitutes for a stronger JSettlers or
  Catanatron adapter.
- Do not use Q advantage for policy updates yet. The latest strict gates show
  that Q/VRPO-style branches can improve a surface leg while regressing
  tactical or rollout-search opponents.
- Graph-history cannot be initialized from the flat champion checkpoint. New
  graph lanes should either start from a graph checkpoint or use reanalysis
  pretraining directly.
- The legacy `tools/league_orchestrator.py` defaults have been changed away
  from Q-mixed `league_vrpo` templates. Default long-run templates are now
  champion-initialized, Q-off, and focused on adaptive EMA, search EMA DAgger,
  and all-seat low-KL pressure. This prevents accidental VM launches from the
  old `s4806` champion or uncalibrated Q-policy mixing.
- Recent policy-gradient work is a reason to keep PPO/regularized self-play as
  the baseline, not to keep random PPO fanout. The failure mode is weak
  population selection and noisy gates; fix those before adding more algorithm
  surface area.
- Recent four-player Big 2 self-play work is closer to Catan than two-player
  poker papers: staged curricula, opponent diversity, and robust evaluation are
  the transferable pieces. Do not copy a two-player zero-sum exploitability
  objective into Catan.

## Long-Term Algorithm Plan

1. Immediate: keep the `s9837`/`s9840`-`s9847` strict-repair VM sweep running
   and do not add local compute. Once non-init iter checkpoints appear, launch
   remote strict gates in small batches with fresh poll, remote-grade-status,
   and local-controller-status preflight.
2. Promotion policy: require candidate-versus-`current_best_s9752_iter0002`
   strict improvement and no per-opponent regression on heuristic,
   `jsettlers_lite`, `catanatron_value`, or `value_rollout`. Aggregate wins
   with a failed tactical leg are rejects.
3. Next code branch: add resource/trade-plan auxiliary targets. Distill
   JSettlers-lite plan features into model-visible labels: selected build
   target, resource deficits for that target, trade progress score, discard
   priority, and accept/reject trade value. Use these as imitation/auxiliary
   losses, not as direct promotion metrics.
4. Q path: run `pfsp_q_calibration` only as critic calibration. It may train
   Q and Expected-SARSA targets, but `--q-advantage-mix` remains `0.0` until
   strict gates prove no tactical regression. VRPO is useful here as variance
   reduction, not as a license to let uncalibrated Q choose policy actions.
5. Search/reanalysis path: build DAGS-style midgame-start data only after safe
   state serialization exists. Use it to diversify starts and teacher labels,
   not as a promotion metric. Prioritize disagreement states where the network
   disagrees with JSettlers-lite/resource-plan and value-rollout teachers.
6. Population path: use the payoff ledger for PFSP scheduling now; move toward
   Global-PSRO-style population expansion only after we have several
   non-regressing frozen policies. The selection objective should minimize
   worst-leg exploitability, not just weighted win rate.
7. Representation path: graph/history models need their own initialized
   lineage or reanalysis pretraining. Do not initialize a graph-history branch
   directly from the flat `s9752` checkpoint. The graph lane becomes serious
   only when it clears strict non-regression.
8. Baseline path: keep `jsettlers_lite` as the cheap tactical gate, but add a
   stronger external JSettlers/Catanatron adapter before claiming strength
   beyond internal bots.

### Research-Driven Next Algorithm Changes

These should be code/tooling changes before another broad launch:

- Keep PPO/regularized self-play as the baseline. The 2026 policy-gradient
  reevaluation supports PPO-style methods as competitive in imperfect
  information games, and the current failures are selection/evaluation
  failures more than optimizer failures.
- Strengthen EMA/old-policy regularization, but keep it tied to opponent
  non-regression. EMAgnet supports an adaptive EMA regularizer in large-game
  self-play; in this repo it should remain an anti-regression stabilizer, not
  a promotion reason by itself.
- Convert the payoff ledger into PFSP opponent sampling. The current weak-leg
  priority says the next sampler should over-sample `catanatron_value` and
  `jsettlers_lite`, with a smaller but nonzero `value_rollout_search` share.
- Add resource-plan auxiliary labels before another generic PPO sweep. The
  latest rejects repeatedly fail trade/resource planning, so model-side
  support for build-target deficits and trade value is more likely to move the
  gate than more unstructured self-play.
- Add DAGS-style midgame starts only after adding a start-state flag. Catan
  has long horizons and sparse terminal reward, so intermediate starts are
  valuable, but the DAGS bias warning applies directly to hidden-resource
  games.
- Turn the payoff ledger into a Global-PSRO-style scheduler only after we have
  enough non-regressing population members. Until then, use PFSP/worst-leg
  sampling and strict champion gates.
- Build Catan-specific tactical targets. The Catan cross-dimensional network
  result and the HexMachina/Catanatron result both point toward richer
  state/action structure and tactical modules, not just more flat PPO games.
- Do not let research outrun gates. Big 2/PPO evidence, PPO-EMAg, VRPO, DAGS,
  PSRO, and ReBeL each map to a tool here, but the acceptance criterion remains
  VM strict-gate improvement over `current_best_s9752_iter0002` with no
  JSettlers/value-rollout regression. The next long-term work should improve
  the opponent sampler, tactical gates, and search/reanalysis data pipeline
  before expanding model size.

## Longer-Term Roadmap

1. Harden the gate: strict VM grading, tactical modules, common seeds,
   value-rollout/JSettlers non-regression, and no promotion from noisy dev-only
   wins.
2. Keep every VM busy, but with role separation: training slots produce
   checkpoints, gate slots compare against `current_best_s9752_iter0002`, and
   local work only plans, polls, analyzes, and edits tooling.
3. Turn the current hand-written opponent suite into explicit tactical gates:
   robber target, discard choice, city/settlement priority, monopoly, maritime
   trade, domestic trade accept/reject, and road race blocking.
4. Reanalysis v2: regenerate sparse search data from whole games and
   disagreement-prioritized states, not just shallow samples. Use search target
   policy, value vector, root action scores, and metadata in every row.
5. Graph-history parent: produce a non-regressing graph checkpoint, then use it
   as the parent for all graph/search lanes instead of starting graph models
   from scratch.
6. Critic/Q calibration: separate value/Q replay phase before Q-policy mixing.
   Required diagnostics before any Q-policy lane: held-out Brier/ECE,
   `q_chosen_return_corr`, legal-action Q spread, advantage sign agreement, and
   no regression in strict VM gates.
7. Seat/phase advantage normalization: normalize returns by seat, phase, and
   prompt family to reduce Catan variance before PPO advantage normalization.
8. Population league: frozen champions, recent snapshots, heuristic/value/search
   bots, and hand-coded exploiters. Use payoff-table/PFSP sampling first; add
   AlphaRank or similar empirical-game analysis after enough cross-play data
   exists.
9. Belief-aware search: public-history belief particles for hidden resources and
   dev cards, root-sampled search, then search distillation into graph-history
   policy.
10. Larger baseline ladder: keep `jsettlers_lite` as a cheap gate, but add a
   real JSettlers/Catanatron adapter before claiming strength beyond heuristic
   bots.
11. Colonist-facing work only after model strength is real, permissioned, and
    separated from training infrastructure.

## Research Anchors

- GAE/VRPO in imperfect-information self-play: use Q-style variance reduction
  as an evidence-generating diagnostic, but do not turn it into policy
  advantage mixing until Catan gates pass:
  https://arxiv.org/abs/2605.19235
- EMAgnet: supports parameter-space EMA regularization for policy-gradient
  self-play; maps directly to `--ema-policy-kl-coef` plus
  `--ema-policy-decay` in current PPO branches:
  https://arxiv.org/abs/2606.23995
- GARIP: moving-reference regularization reduces last-iterate cycling in
  self-play; maps to moving-reference/EMA anchors rather than fixed-only
  anchors:
  https://arxiv.org/abs/2606.22688
- Regularized policy optimization for board games: reverse-KL regularization,
  entropy regularization, and model-free board-game experiments motivate the
  `pfsp_klent_control` ablation. Keep it as a control branch and require
  normal strict non-regression before any promotion:
  https://arxiv.org/abs/2602.10894
- Global PSRO: use population expansion only after we have multiple useful
  frozen policies. The transferable idea is selecting responses that improve
  the whole population/worst-leg coverage, not launching more one-off PPO
  branches:
  https://arxiv.org/abs/2605.28273
- Generals.io self-play: strong practical evidence for high-throughput
  imperfect-information self-play with strict held-out evaluation and
  search/sample-filtering ideas:
  https://arxiv.org/abs/2606.23348
- ZeusAI / 7 Wonders Duel: AlphaZero-style search plus transformer policies can
  work for modern board games; useful as a search-distillation model, not a
  reason to abandon Catan-specific gates:
  https://arxiv.org/abs/2406.00741
- Deep Catan / neural MCTS: search plus neural policy/value targets is a better
  fit than pure PPO for long-horizon Catan decisions.
- ExIt: use search to produce stronger targets, then distill those plans into
  the network instead of expecting the network to discover every plan directly:
  https://arxiv.org/abs/1705.08439
- Cross-dimensional Catan neural networks: board graph/hex-edge-vertex
  representation is better aligned than flat MLP inputs, and prior Catan work
  reports outperforming JSettlers with this direction:
  https://arxiv.org/abs/2008.07079
- Catanatron: use it as the high-throughput external Catan simulator/player
  reference. Its project goal is explicitly strong Catan bot benchmarking at
  scale, so a real adapter is a better medium-term baseline than only internal
  `catanatron_value` approximations:
  https://github.com/bcollazo/catanatron
  https://docs.catanatron.com/
- JSettlers2: keep it as the historical rule-based/robot-opponent benchmark
  line. Short-term `jsettlers_lite` gates are useful, but the roadmap should
  include a real adapter before claiming strength over JSettlers:
  https://github.com/jdmonin/JSettlers2
- ReBeL/public-belief search: the exact theorem is two-player zero-sum, but the
  engineering lesson still applies to Catan: train on public-belief/search
  targets, not just sampled hidden-state rollouts:
  https://arxiv.org/abs/2007.13544
- POMCP in Catan: hidden-resource belief search is a practical fit.
- PPO/regularized policy optimization and recent VRPO-style work: current
  KL/entropy regularization and Q/variance-reduction diagnostics are relevant,
  but Q must be calibrated before it drives policy updates.
- 2026 EMAgnet / PPO-EMAg: parameter-space EMA regularization is a good match
  for the active `s9764`, `s9765`, `s9766`, and `s9768` style lanes because it
  keeps PPO regularized toward a moving policy target rather than wasting
  pressure on dominated actions:
  https://arxiv.org/abs/2606.23995
- 2026 VRPO / Q-boosting: the useful idea is Expected-SARSA-style variance
  reduction with a centralized action-value critic; current VM results say to
  keep it behind calibration gates, not promote it blindly:
  https://arxiv.org/abs/2605.19235
- 2026 DAGS: data-augmented starts suggest a concrete later upgrade for Catan:
  start training/search from mined midgame states instead of always from game
  start, once we have reliable replay state serialization:
  https://arxiv.org/abs/2605.14379
- 2026 Global PSRO: population quality and exploitability-aware expansion are
  the right inspiration for our league controller, but the paper is two-player
  zero-sum, so Catan should adapt only the population-selection machinery:
  https://arxiv.org/abs/2605.28273
- AlphaStar/PSRO/AlphaRank: league, exploiters, empirical payoff tables, and
  opponent-aware training are the right long-term shape for a strong
  imperfect-information multiplayer board-game agent:
  https://deepmind.google/blog/alphastar-grandmaster-level-in-starcraft-ii-using-multi-agent-reinforcement-learning/
  https://arxiv.org/abs/1903.01373
  https://arxiv.org/abs/2403.02227
- Catan negotiation RL is useful later, but not the next bottleneck. First make
  no-trade/bank-trade policy strong and tactically robust:
  https://arxiv.org/abs/1511.08099
- Reevaluating policy gradient methods for imperfect-information games: useful
  counterweight to assuming PPO is fundamentally wrong; our data says the
  bottleneck is population/gating discipline:
  https://arxiv.org/abs/2502.08938
- Big 2 four-player imperfect-information self-play: closer to Catan than
  two-player poker for curriculum/opponent diversity lessons:
  https://arxiv.org/abs/2605.28863
- GARIP running-average moving reference: supports the current direction of
  regularizing self-play toward a moving reference policy instead of relying on
  naive last-iterate PPO. For Catan, the concrete branch is stronger
  EMA/old-policy reference control, not removing KL anchors:
  https://arxiv.org/abs/2606.22688
- Generals.io self-play under strong imperfect information: useful operational
  benchmark for hidden-information tactical training at scale. Catan takeaway:
  keep VM-side self-play saturated, but make promotion depend on tactical
  anti-regression gates rather than raw training reward:
  https://arxiv.org/abs/2606.23348
- AlphaZero-style board-game transfer remains relevant for search
  distillation; the 7 Wonders Duel work is a recent example outside chess/go.
  For Catan, use this as support for value-rollout/search-teacher distillation
  after the PPO population is stable:
  https://arxiv.org/abs/2406.00741
